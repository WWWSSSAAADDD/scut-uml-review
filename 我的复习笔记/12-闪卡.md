### D. GRASP 九大模式

| # | 正面 | 背面 |
|---|------|------|
| D-1 | Information Expert | Assign a responsibility to the class that has the information needed to fulfill it. |
| D-2 | Creator | B creates A if B contains/aggregates/closely-uses A or has the data to initialize it. |
| D-3 | Controller | Assign the responsibility for handling a system event to a class representing the overall system, a use case scenario, or a role. |
| D-4 | Low Coupling | Assign responsibilities so that coupling remains low. |
| D-5 | High Cohesion | Assign responsibilities so that cohesion remains high. |
| D-6 | Polymorphism | When alternatives vary by type, assign the responsibility using polymorphic operations. |
| D-7 | Pure Fabrication | Assign a highly cohesive set of responsibilities to an artificial class that does not represent a domain concept. |
| D-8 | Indirection | Assign the responsibility to an intermediate object to mediate between other components. |
| D-9 | Protected Variations | Identify points of predicted variation and create a stable interface around them. |
